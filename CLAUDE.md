# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

This is the **OpenClaw main agent** orchestration project — a "full automatic development orchestration" scenario. It uses **Claude Agent SDK (TypeScript)** to orchestrate Claude Code, integrates with **gstack** workflow, and drives development via **GitHub/GitLab Issues** following SDD + DDD + TDD patterns.

**Key Reference**: `docs/SKILL-SPEC.md` is the authoritative implementation spec for the OpenClaw Skill and TypeScript SDK dispatch layer. The `docs/prompt/` directory contains prompt templates (coding_prompt.md, initializer_prompt.md).

## Package Manager

**pnpm only** — do not use npm or yarn as the default installation method. The `pnpm-lock.yaml` must be committed alongside any dependency changes. CI uses `pnpm install --frozen-lockfile`.

Node version requirement: `>=20` (check with `node -v`).

## Development Commands

```bash
pnpm install          # Install dependencies
pnpm test             # Vitest unit tests (mock + dispatch logic, no real claude/API keys)
pnpm lint             # ESLint
pnpm format           # Prettier write
pnpm format:check     # Prettier check
pnpm typecheck        # tsc --noEmit
pnpm build:sdk        # Build packages/sdk → dist/
```

For running a single test file, use Vitest directly: `pnpm vitest run src/path/to/test.test.ts`

## CI

- **PR CI** (`.github/workflows/ci.yml` on push/PR to main/master): installs, lint, format:check, typecheck, test — no real API keys needed.
- **Nightly/Manual** (`.github/workflows/claude-integration.yml`): real `claude` smoke tests via `ANTHROPIC_API_KEY` secret. Run locally with `pnpm integration:smoke`.

## Architecture Notes

- **Orchestrator role**: Does not write business code — only manages Issues, starts/stops tasks, summarizes status.
- **Coding sessions**: Allow business code changes, tests, PR creation.
- **Memory truth source (v1)**: Repo-internal memory + Issue/comments, not OpenClaw built-in context magic.
- **Failure handling**: Auto-retry within same work unit; 5 consecutive failures notify human. Each new SDK session resets the counter. Loop implemented in SDK outer layer, not Ralph Stop Hook.
- **PR workflow**: `/review` → `/ship` → `/qa` (gstack slash conventions). **Close Issue after PR merges successfully**.
- **Branching**: Each worktree = one branch = one PR. Fixed naming convention (e.g., `issue-<id>-<slug>`) must avoid multi-session branch conflicts.
- **gstack integration**: Global install; skill lists for project setup are generated by tech stack context.
